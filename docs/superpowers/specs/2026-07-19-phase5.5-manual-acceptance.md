# Phase 5.5 — Viewer Editor Panels: Manual Acceptance

**Date:** 2026-07-19
**Status:** Pending manual acceptance
**Spec:** `2026-07-19-phase5.5-viewer-editor-panels.md`

This checklist covers the interactive editor loop that Phase 5.5 adds to
MatterViewer. It exercises every new panel, the selection model, gizmo,
simulation toolbar, console, and cross-cutting integration points. Automated
headless tests (world-definition, console-log, selection-set) complement but
cannot replace this list — they verify data models, not the UI wiring.

## Setup

1. Build `MatterViewer` Windows target: `make -C MatterViewer windows` (with
   TEMP/TMP set per CLAUDE.md).
2. Launch `viewer.exe` from the `MatterViewer/` working directory.
3. Connect to **PhysicsPlayground** (or any world with authored ECS entities
   and baked geometry).
4. Wait for bake to complete (console shows "Bake finished").

## Panel layout

- [ ] **Default layout**: Toolbar strip at top, Scene tree on the left,
      Properties panel on the right, Console at the bottom, viewport fills
      the remaining center area.
- [ ] **Panel persistence**: Drag a panel to a new position, close and reopen
      the viewer. Panel positions restore from `imgui.ini`.
- [ ] **Resize**: All panels resize with the window; no horizontal overflow.

## Scene tree

- [ ] **Baked roots**: `[Root]` nodes appear for each module in the part graph
      snapshot.
- [ ] **ECS entities**: `[Entity]` nodes appear for each authored entity
      (in Edit mode).
- [ ] **Filter**: Type in the search box. Only matching nodes remain visible;
      clearing the box restores the full tree.
- [ ] **Filter mode tabs**: "All", "Entities", "Baked" filter buttons work.
- [ ] **Single click**: Clicking an entity row selects it (Properties panel
      updates, selection outline appears in viewport).
- [ ] **Ctrl+click**: Toggles individual items in/out of the multi-selection.
- [ ] **Shift+click**: Extends selection range.
- [ ] **Click on baked root**: Selects the root; Properties shows read-only
      baked root info card (module name, hash, source path, params JSON).

## Properties inspector

- [ ] **Auto-generated widgets**: Selecting an entity shows its components
      (LocalTransform, RigidBody, etc.) with editable fields.
- [ ] **Float3 fields**: DragFloat3 widgets for translation, scale, velocity.
- [ ] **Quaternion fields**: DragFloat4 widget for rotation.
- [ ] **Bool fields**: Checkbox for boolean component fields.
- [ ] **PartInstance editor**: Shows part picker popup with searchable module
      list; clicking a module assigns the part.
- [ ] **RigidBody editor**: Wake/Apply Impulse/Set Velocity/Teleport buttons
      with Float3 input fields.
- [ ] **SectorStreaming editor**: Attach/Remove, radius drag, Follow Camera
      toggle, Seed+Regenerate button.
- [ ] **Baked root card**: Read-only info for `[Root]` selections — module,
      source_path, hex hash with Copy button, params JSON, child count, "Open
      Source" button (opens the .js file in the system editor).
- [ ] **Multi-select**: Selecting multiple entities shows common components;
      editing a field applies to all selected.
- [ ] **Edit disabled during Play**: Property widgets are disabled/read-only
      when SimulationMode is Play.

## Viewport picking

- [ ] **Click entity in viewport**: Clicking an entity mesh in the 3D viewport
      selects it (scene tree highlights, properties update).
- [ ] **Click baked geometry**: Clicking static baked geometry selects the
      corresponding baked root.
- [ ] **Click empty space**: Clicking empty viewport clears selection.

## Selection outline

- [ ] **AABB wireframe**: Selected entities display a wireframe AABB overlay
      in the viewport.
- [ ] **Multi-select**: All selected entities show outlines simultaneously.
- [ ] **Deselect removes outline**: Clearing selection removes all outlines.

## Transform gizmo

- [ ] **Default translate**: Gizmo appears in translate mode when an entity is
      selected.
- [ ] **Hotkeys**: `G`/`T` = translate, `R` = rotate, `S` = scale.
- [ ] **Drag translate**: Dragging gizmo arrows moves the entity; Properties
      panel translation updates in real time.
- [ ] **Drag rotate**: Rotate gizmo ring rotates the entity; quaternion
      updates.
- [ ] **Drag scale**: Scale gizmo handles scale the entity.
- [ ] **Multi-select translate**: Translating with multiple entities selected
      fans the delta to all selected entities.
- [ ] **Gizmo suppresses camera**: Dragging the gizmo does not orbit the
      camera.
- [ ] **Hidden during Play**: Gizmo does not appear in Play mode.
- [ ] **Hotkeys ignored in text fields**: Typing in a text/search field does
      not change gizmo mode.

## Simulation toolbar

- [ ] **Play**: Clicking Play transitions to Play mode; viewport border turns
      green; toolbar shows "PLAYING".
- [ ] **Pause**: Clicking Pause during Play transitions to Pause mode; viewport
      border turns amber; toolbar shows "PAUSED".
- [ ] **Step**: Clicking Step during Pause advances one physics step.
- [ ] **Stop**: Clicking Stop restores the Edit-mode snapshot, clears
      selection, returns to Edit mode; viewport border disappears.
- [ ] **Space toggle**: Pressing Space toggles Play/Pause (not during text
      input).
- [ ] **Space no-repeat**: Holding Space does not rapid-fire toggle.
- [ ] **Button enable/disable**: Play greyed in Play; Pause greyed outside
      Play; Step greyed outside Pause; Stop greyed in Edit.

## Runtime classification

- [ ] **[Runtime] tag**: During Play, entities spawned by the simulation
      (not in the Edit-mode snapshot) show `[Runtime]` tag in the scene tree.
- [ ] **[Entity] tag**: Authored entities keep `[Entity]` tag during Play.
- [ ] **After Stop**: All `[Runtime]` entities disappear; tree returns to
      Edit-mode population.
- [ ] **Multiple Play/Stop cycles**: Tags remain correct after repeated
      Play → Stop transitions.

## Console/log panel

- [ ] **Bake messages**: Bake progress and completion messages appear.
- [ ] **Severity icons**: Info/Warning/Error entries have distinct visual
      treatment.
- [ ] **Filter**: Severity filter buttons and text search work.
- [ ] **Clear**: Clear button empties the log.
- [ ] **Auto-scroll**: New messages scroll the console to the bottom.

## Context menus (right-click)

- [ ] **Entity row**: Right-click shows Focus / Add Child / Duplicate / Delete.
- [ ] **Focus**: Moves camera to frame the entity's AABB.
- [ ] **Add Child**: Creates a child entity visible under the parent in the
      tree.
- [ ] **Duplicate**: Spawns an offset copy of the entity.
- [ ] **Delete**: Removes the entity from tree and viewport.
- [ ] **Baked root row**: Right-click shows Focus / Open Source.
- [ ] **Destructive items disabled during Play**: Delete, Duplicate, Add Child
      are greyed out while in Play mode.
- [ ] **F hotkey**: Pressing F focuses the camera on the current selection.
- [ ] **Delete hotkey**: Pressing Delete removes all selected entities (Edit
      and Pause only).

## Pause-edit interaction

- [ ] **Edit during Pause**: Changing a property while paused applies the
      change immediately (visible after a Step).
- [ ] **Discard hint**: A yellow "changes made in Pause mode are discarded on
      Stop" note appears in the Properties panel the first time Pause mode is
      entered with a selection.
- [ ] **Stop discards**: Edits made during Pause are lost after Stop (snapshot
      restore wins).
- [ ] **Resume keeps**: Edits made during Pause survive Play resume.

## World switch and reload

- [ ] **Switch clears selection**: Switching worlds clears both SelectionSet
      and EditorModel selection.
- [ ] **Switch resets sim**: SimulationControl returns to Edit mode on world
      switch.
- [ ] **Reload clears selection**: Reloading the current world clears
      selection and resets sim state.
- [ ] **No stale state**: After switching worlds, the scene tree shows the new
      world's entities; no ghost entries from the previous world.

## Cross-cutting

- [ ] **Hotkey vs. text input**: Typing `G`, `R`, `S`, `F`, `Delete`, `Space`
      in the scene tree filter box or a Properties text field does NOT trigger
      gizmo mode changes, focus, delete, or play/pause. ImGui's
      WantTextInput/WantCaptureKeyboard guards are active.
- [ ] **No console errors**: The console log shows no unexpected Error entries
      during the full acceptance flow.
- [ ] **Linux target compiles**: `ui_linux.cpp` and `main_linux.cpp` compile
      (link may fail due to missing Linux libs; compilation is the gate).
